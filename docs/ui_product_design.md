# AlphaSeeker UI 与产品设计（当前版）

## 1. 当前信息架构

- 主页：扫描参数、推荐列表、执行诊断、最近报告
- 持仓：交易操作（买卖）、实时估值、快照保存、持仓对比、历史流水
- 观察评估：历史推荐样本的胜率与收益统计
- Settings：模型配置与用户环境变量
- 测试功能：LLM 连通性测试

## 2. 数据持久化

- `data/reports/scan_*.json`：每次扫描结果与诊断
- `data/alphaseeker.db`：
	- holdings：当前持仓
	- trades：交易流水（买卖、数量、价格、手续费、时间、备注）
	- holdings_snapshots：持仓快照（用于历史对比）

## 3. 产品目标匹配度

已覆盖：
- 股票推荐
- 当前持仓数据
- 交易操作与卖出记录
- 持仓历史快照对比
- 持久化数据
- 失败诊断与可追溯

仍需补齐（下一阶段）：
- 交易流水（买卖记录）与净值曲线
- 推荐命中归因（行业/主题/风险暴露）
- 报警中心（异常数据源、策略失效）
- 权限与审计（多用户/操作日志）

## 4. UI 下一步建议

- 首页增加“今日信号摘要卡片”（推荐数、通过率、失败环节）
- 持仓页增加“调仓建议”与“风险暴露”
- 观察评估页增加周/月切换与分组统计
- 设置页增加“数据源健康状态”可视化
